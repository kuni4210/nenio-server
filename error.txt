- NotFoundException
용도: 요청한 리소스를 찾을 수 없을 때 발생시킵니다. 데이터베이스에서 결과를 찾지 못하는 등의 상황에 사용됩니다.
상태 코드: 404 (Not Found)

- ConflictException
용도: 서버에서 요청을 수행하는 중에 데이터 충돌이 발생했을 때 사용됩니다. 주로 데이터베이스에 중복된 데이터를 추가하려고 할 때 발생시키며, 데이터 충돌이 발생하여 요청을 처리할 수 없음을 나타냅니다.
상태 코드: 409 (Conflict)

- ServiceUnavailableException
용도: 서버가 일시적으로 요청을 처리할 수 없는 상태일 때 사용됩니다. 주로 서버 유지 보수, 과부하, 또는 외부 서비스와의 연결 장애 등이 발생했을 때 발생시킵니다.
상태 코드: 503 (Service Unavailable)

- GatewayTimeoutException
용도: 서버가 게이트웨이나 프록시 역할을 하고 있는 경우, 게이트웨이나 프록시로부터 제때에 응답을 받지 못했을 때 발생시킵니다.
상태 코드: 504 (Gateway Timeout)

- UnauthorizedException
용도: 인증되지 않은 사용자가 보호된 리소스에 접근하려고 할 때 발생시킵니다. 주로 인증이 필요한 엔드포인트에 인증되지 않은 사용자가 접근하려고 할 때 사용됩니다.
상태 코드: 401 (Unauthorized)

- ForbiddenException
용도: 인가되지 않은 사용자가 리소스에 접근하려고 할 때 발생시킵니다. 주로 인증은 되었지만 해당 리소스에 접근 권한이 없는 경우에 사용됩니다.
상태 코드: 403 (Forbidden)

- InternalServerErrorException
용도: 서버 내부 오류로 인해 요청을 처리할 수 없는 경우에 발생시킵니다. 예상치 못한 예외 상황이 발생한 경우에 사용됩니다.
상태 코드: 500 (Internal Server Error)

- BadRequestException
용도: 클라이언트가 잘못된 데이터를 보냈거나 요청이 올바르지 않을 때 발생시킵니다. 예를 들어, 요청 파라미터가 필요한 형식에 맞지 않거나, 필수 필드가 누락되었을 때 사용할 수 있습니다. 데이터의 유효성 검사에서 문제가 발생하는 경우에도 이 예외를 활용할 수 있습니다.
상태 코드: 400 (Bad Request)
